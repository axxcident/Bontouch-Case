Bontouch Case

This test aims to validate your technical skills and code quality.

Please read the instructions carefully before starting on the case.

The Mission

We would like you to create a web application for displaying users and their photo albums.

You will most likely need 4-8 hours to finish this task, and we suspect that you will spend most of that time with the album.

Views

The application should consist of three views:

  • Users
  • User details
  • Album

Each view should be represented in a top bar as a breadcrumb.

Users

The view should contain of two lists; one listing the favorited users and one listing the rest. You should be able to favorite a user by tapping the star in the upper right corner. The list of favorited users should be persisted in the browser.

User details

Displaying the user's contact information and listing the user's albums.

Album

Listing the photos in the album and showing them in a photo viewer when pressed.

Please give the bonus tasks a try as well, even if you dont solve them perfectly

Bonus: When the photo viewer is opened/closed, it should animate from/to the position of the screen that the thumbnail is positioned.

Bonus #2: Add more functionality to the photo viewer, for example being able to navigate photos using buttons or keyboard arrows.

Bonus #3: Make the closing animation work when current photo (when closing) is different from the one opened.

It should be possible to refresh the browser and stay on the same page.

Techniques

Use a commonly used framework like React or Vue, or ask us if you want to use something different.

CSS preprocessors and HTML templating engines are welcomed to use.

Third-party libraries should be kept at a minimum. The photo viewer should not use any external library.
Although, common third-party libraries as react-router, vue-router and such that are tightly coupled with the framework should be used.

You are also allowed to use commonly used state management libraries if you find them useful.

If you're unsure about which libraries/frameworks that are okay to use, feel free to ask.

Resources

API

To populate the application with data you will be using the already existing JSONPlacehoder API.

You do not need to create any backend to solve the case.

Heads up: The route structure under "Routes" can be applied on the other endpoints as well.
